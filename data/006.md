## Broken Access Control - IDOR

I discovered an IDOR (Insecure Direct Object Reference) within the OWASP Juice Shop. It allows me access to something I do not have the correct privileges to access.

In this case I found that I could change which users cart I have access to in a couple of different ways.

First I went through Burp Suites repeater and changed my GET Request to a different number than my own. We are using the API endpoint of /rest/basket/x.

![BURP Idor](<../images/2026-03-04 - Idor burp.png>)

And the second way I could change the user cart was through the regular dev tools in the web browser. Here I changed the key "bid" (basket id I'm guessing) from 4 to 1 and updated the page.

![bid](<../images/2026-03-04 - Broken access control - session storage.png>)

To make sure this happens we need to make sure that we implement code that makes sure to verify ownership and authentication of assets serverside and not client side. We should return a 403 when there's a mismatch.
